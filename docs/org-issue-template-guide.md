# Issue Template Guide

## Purpose

Every issue should be structured enough that a new contributor or agent can start without re-asking the same questions.

## Issue Types

- `feature`
- `bug`
- `research`
- `docs`
- `chore`

## Required Fields

- summary
- background
- expected outcome
- scope
- acceptance criteria
- risks
- links

## Feature Issue Template

Use this when adding a feature or workflow.

- What is the user problem?
- What should the new behavior be?
- What must stay unchanged?
- What files are likely affected?
- What counts as done?

## Bug Issue Template

Use this when fixing a broken behavior.

- What happened?
- What should have happened?
- How can it be reproduced?
- What changed recently?
- What is the safest fix?

## Research Issue Template

Use this when exploring model behavior, prompt strategy, or AI routing.

- What are we trying to learn?
- What options are being compared?
- What is the decision criterion?
- What sample size or test flow is enough?
- What result would change the plan?

## Documentation Issue Template

Use this when improving docs or operational guidance.

- Which doc needs updating?
- Which behavior changed?
- Which template or policy is affected?
- What should future contributors know?
